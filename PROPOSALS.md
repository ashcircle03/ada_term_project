# FruitsFamily 제품 제안 — 셀러 유동성 개선 (내부 PM 원페이저)

> 근거: 본 분석(`INSIGHTS.md`, `results/*.json`, 표본 284,654 매물 / 11,095 셀러).
> 모든 효과 추정은 **관측 데이터 기반**이라 역인과 위험이 있다 → 각 제안은
> "실험(A/B)으로 검증할 가설"로 제시한다. 수치는 그 방향성의 근거다.
>
> **기존 기능(사용자 확인):** 개인화 추천 피드(구매자측)·끌어올리기/부스팅은 있음.
> 별도 셀러 대시보드는 **없고**, 상품 페이지에 **조회수·하트(위시)수만** 노출. 등록 시점
> **가격 제안/시세는 없음**(셀러가 직접 검색). → 제안은 이 제약에 맞춰 신규/개선을 구분.

## 핵심 재프레이밍 (왜 지금까지의 접근이 안 통하나)

1. **유동성이 문제다.** 매물 78.5%가 미판매, 미판매의 55%가 90일+ 방치, 셀러 16%는 0건 판매.
2. **현재 등록 안내("사진 많이·설명 길게")는 데이터가 반박한다.** 가격대를 통제해도
   사진 ≤2장(26%)이 6+장(19%)보다 더 팔리고, PSM ATT는 **−2.2pp**(가이드 준수가 오히려 약불리).
3. **셀러가 움직일 수 있는 여지는 작다 — 전환 예측력의 13%뿐**(구조 87%). 고가품일수록 더 무력.
   → 레버는 "셀러에게 더 일하라"가 아니라 **가격 정확도·묵은재고 개입·매칭**에 있다.

## P2 · 등록 시 동종 가격 밴드 (최우선, 신규)

**문제:** 가격 가이드가 등록 화면에 없어 셀러가 직접 검색해야 한다. 빈티지 1점물은
시세가 불투명해 미스프라이싱이 흔하고, 이는 유동성 위기의 직접 원인.

**데이터 근거:**
- 상대가격(동종 대비)은 셀러 통제가능 레버 중 **드물게 판매와 양(+)의 연관**(H1 OR 1.025, p<.001).
- 가격 레버는 **저가일수록 유효**(H2 통제가능 AUC: 저가 0.65 → 고가 0.57).
- `brand × category_l1 × condition` 그룹으로 **96.7% 매물에 가격 밴드 산출 가능**(이미 구현됨).

**메커니즘 (등록 폼):** 브랜드·카테고리·상태 입력 즉시
"비슷한 매물 가격대: **p25–p75 = X–Y원** (중앙값 Z). 입력가는 동종 대비 **+N%**" 표시.
표본 부족 그룹(n<5)은 `brand×category`로 폴백, 그래도 부족하면 비표시.

**성과지표 / A/B:** 단위=신규 매물. 처치=가격 밴드 노출. 1차=60일 내 sell-through,
2차=가격 수정율·등록 이탈율(가드레일). 미스프라이싱(밴드 밖 등록) 비율 변화 추적.

**한계:** 상대가격의 양의 연관은 약하고 선택편의 가능(좋은 변형이 비싸고 더 팔림).
→ "싸게 내라"가 아니라 **정확히·투명하게**가 메시지. 효과는 A/B로 확정.

## P1 · 일반 안내 → 조건부 안내 (등록 시점, 개선)

**문제:** 모두에게 동일한 "사진 많이·설명 길게"는 관측상 지지되지 않으며(H1, 인과 단정 불가), 노력 기여는
13%(H2). 특히 고가 셀러에겐 헛된 기대.

**데이터 근거 (통제 후 OR):**
- 효과 **없음/역방향**: 사진 3–5장 0.92, 6+ 0.86 / 설명길이 0.97 / 실측 0.75 · 하자고지 0.89 · 관리법 0.81
  (자세할수록 안 팔림 = 어려운 매물에 설명 집중되는 역인과)
- **그나마 양(+)**: 사용맥락 **1.13** · 소재 1.04 · 합리적 상대가격 1.025

**메커니즘:** 등록 안내 문구를 교체.
- 공통: "사진 수·글자 수보다 **정확한 가격(P2)·왜 내놓는지(사용맥락)·정직한 상태**가 판매를 좌우합니다."
- 세그먼트: **고가·브랜드 매물** → "표현보다 **정품성·가격**이 결정적입니다"로 분기
  (H2: 고가품 통제가능 AUC 0.57).

**성과지표 / A/B:** 처치=새 안내 카피. 1차=sell-through, 2차=등록 완료율·설명 작성 행동 변화.

**기존 대비:** 현재 셀러가 보는 건 상품페이지의 조회/하트 숫자뿐 — 이건 **등록 시점의 처방적 안내**.

## P3 · 묵은재고 개입 — 가격 실험 + 찜 오퍼 (차별: 끌어올리기 ≠ 재노출)

**문제:** 미판매 55%가 90일+ 방치, 90일 이후 전환 정체 = **유동성 절벽**. 끌어올리기는
*재노출*만 할 뿐 — 90일+ 미판매의 **74%는 이미 충분히 노출**됐는데도 안 팔린다(재노출 무용).

**데이터 근거 (`results/p3_staleness.json`):**
- ⚠️ **전제 반박:** "동종 중앙값까지 인하"가 통하려면 동종군에서 싼 게 더 팔려야 하나,
  가격위치별 sell-through는 **평평**(저가 19.2% ~ 고가 19.9%), 판매/미판매 백분위 0.519/0.509.
  → **단순 인하의 효과는 관측상 미지지**(매력적 변형이 비싸도 팔리는 내생성). 효과는 A/B로만 확정.
- 규모(90일+ 미판매 12.3만): **49%가 동종 중앙값 초과**(중앙값까지 −37.5%), **93%가 찜 보유**.
- 진단: 90일+의 **74%가 "충분노출+미판매"**(가격/오퍼 영역), 26%만 저노출(매칭/P4).

**메커니즘 (두 arm 분리):**
- **arm A · 가격 실험:** 90일+ "충분노출+과가" 매물에 "동종 중앙값까지 −N%" 제안.
  효과 미지 → **이 트리거 자체가 인하의 인과 탄력성을 처음 측정하는 실험**.
- **arm B · 찜 오퍼:** 찜 보유 묵은매물(93%) → 찜한 사람에게 **비공개 할인 오퍼**.
  드러난 수요를 직접 공략, 공개가는 안 내려도 됨(브랜드가치 보호). wishlist 자산 활용.
- 끌어올리기는 **가격/오퍼 조정과 묶일 때만** 추천(단독 재노출 억제).

**성과지표 / A/B:** 단위=90일 도달 미판매 매물. armA 1차=제안 후 30일 전환·인하 수락율;
armB 1차=오퍼 수락→거래. 가드레일=셀러 이탈·평균 판매가(마진)·인기매물 잠식.

## P5 · 상품페이지 조회/하트에 맥락 + 진단 라우팅 (신규)

**문제:** 셀러가 보는 건 상품페이지의 **조회수·하트수(맥락 없는 숫자)**뿐. "조회 50이
많은지", "하트 10개인데 왜 안 팔리는지" 알 수 없어, 안 팔리는 원인을 **자기 탓(사진/설명)**
으로 오인하고 현재의 (틀린) 일반 안내로 빠진다.

**데이터 근거 (퍼널 분석):**
- ⚠️ **조회/하트는 판매를 거의 변별 못 한다** — 조회 5분위 sell-through 18→22%(Q5 하락),
  하트 5분위 20→23%, 하트>0 미판매율 78.3% vs 하트0 80.5%. 즉 **숫자가 커도 안 팔릴 수 있다**
  (engagement≠conversion, H2와 합치). → 맥락 없는 숫자는 셀러를 오도한다.
- **동종(카테고리×가격대×연령) 정규화 시**: 미판매의 **25.2%만 하위25% 미만 = 저노출**
  (발견 문제 → P4), 나머지 **74.8%는 정상 이상 노출됐는데도 미판매**(→ 가격·신뢰, P2/P3).
  전역중앙값 단순비교(50.8% 저노출)보다 정확 — **죽은재고 대부분은 "안 보여서"가 아니라 "봐도 안 사서"다.**

**메커니즘 (상품페이지, 라우팅):** 숫자에 **동종·동연령 정규화 맥락**을 붙이고 *문제를 라우팅*:
- 조회 < 동종 정상범위 → "노출이 적습니다 — 발견 문제(가격·제목·노출), 사진 추가로는 안 풀림"
- 조회 정상 + 미판매 → "충분히 보였습니다 → **가격(P2)·상태 신뢰**를 점검하세요"
- (선택) 셀러 유형(H3 아키타입)과 동형 상위 셀러의 가격대/회전 위치 표시.

**성과지표 / A/B:** 처치=맥락/진단 노출. 1차=진단 후 행동(가격조정·재노출)→전환,
2차=오귀인 감소(불필요한 사진/설명 수정 행동 감소).

**주의:** "성공 셀러가 설명을 덜 씀"은 역인과 → **"적게 쓰라"로 번역 금지**. 또 조회/하트의
약한 변별력 때문에 "숫자 키우기" 목표가 아니라 **원인 라우팅**이 핵심.

**데이터 근거 (H3, 군집 완료):** 3개 아키타입, 전환율 차이 극유의(KW p<1e-76).
전문가형(HHI 0.70) 14.9% 최저 vs 다양화·중저가형 24.2% — 선행 "전문가=가격 프리미엄"과
합치면 **유동성↔가격 trade-off** → 유형 맥락에 함께 노출.

## P4 · 매칭 우선 전략 — 추천피드를 죽은재고로 재타깃 (개선, 실현가능성 입증)

**문제 인식:** 셀러 노력이 13%뿐이라면, 플랫폼의 최대 레버는 "셀러를 더 굴리기"가 아니라
**"있는 매물을 맞는 구매자에게 더 잘 매칭"**이다. 추천피드는 이미 있으나 인기/신규 위주로
보이며, **묵은 재고 해소**에 최적화돼 있지 않다.

**데이터 근거 (`results/p4_taste_network.json`, wishlist 90.3% 커버리지로 확보):**
- ★ **죽은재고 매칭 후보 모수 뒷받침**: 90일+ 미판매 매물의 브랜드를 찜한 고유 셀러 수 **중앙값 190명**,
  **98.3%가 관심자 ≥1명**(인기 상위10 브랜드 제외 시에도 **80%** ≥1명으로 견고). 죽은 재고 대다수에
  **도달 가능한 관심 풀이 존재**(단 brand-level·셀러측 프록시 — 아래 한계 참조).
- 취향은 구조적(브랜드 HHI 중앙값 0.222)이고 **75.5%가 타 셀러 매물 찜**(= 수요 신호).
- 아키타입↔취향 동질성 1.21배(약함) → **아키타입은 매칭 코호트 대용 불가, 위시 그래프 자체가 신호**.

**메커니즘(탐색):** 추천피드 랭킹에 **aging-inventory 가중**을 더해, 해당 브랜드/취향을 찜한
구매자에게 묵은 매물을 노출(또는 P3-armB 비공개 오퍼와 연동). 성과=죽은재고 전환·전체 GMV
(인기매물 잠식 가드레일).

**남은 불확실성:** "브랜드 찜"은 "이 매물을 산다"의 거친 프록시(인기 브랜드 과대). 셀러측
데이터라 추천 *클릭→구매* 전환은 미측정. → 매칭 후보 모수는 입증됐으나 **전환 효과는 A/B 필요**.

## 검증 원칙 (모든 제안 공통)

- 분석은 **관측·단면(`is_sold`, sold_at 없음)**이라 인과 확정 불가 → **반드시 A/B로 검증**.
- 일관되게 나타난 "노력↔판매 음상관"은 **"적게 하라"가 아니라 "노력이 구조적 비매칭을
  못 메운다 + 어려운 매물에 노력이 몰린다(역인과)"**로 해석. 카피·벤치마크에 오역 주의.
- 1차 지표는 일괄 **sell-through(30/60일)**, 가드레일은 셀러 이탈·등록완료율.

## 제안하지 않은 것 (그리고 이유)

- "사진/설명 더 독려" 강화 → H1에서 인과 무근거·역방향이라 **의도적으로 배제**.
- 신규 추천피드/끌어올리기 → **이미 존재** → 신설 대신 재타깃·연동으로 제안.
- 조회/하트 "숫자 키우기" 독려 → 데이터상 판매 변별력 약함 → 숫자 늘리기 대신 **원인 라우팅**으로.
- 셀러 평점·리뷰 기반 제안 → review 테이블 **데이터 0건**이라 불가.
